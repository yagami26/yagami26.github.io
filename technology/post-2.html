noob golang dev - strings are hell, loops are amazing in golang

May 22, 2024, Posted in Technology

I have started practicing the leetcode Programming Skills study plan. To grasp a language quickly, it has progressive set of exercises. I started with a problem where you were given two strings and you had to merge them alternatively, eg - word1 - "Hello" and word2 - "World" then the function should return a concatenated string "HWeolrllod".

Here is the code which I came up with 

package main

import "fmt"

func main() {
    const (
        word1 = "Hello"
        word2 = "World"
    )

    fmt.Println(mergeStringsAlternatively(word1, word2))
}

func mergeStringsAlternatively(word1 string, word2 string) string {

    word1Len := len(word1)
    word2Len := len(word2)

    var mergedString string
    i, j := 0, 0

    for ; i < word1Len && j < word2Len; i, j = i+1, j+1 {
        mergedString += string(word1[i])
        mergedString += string(word2[j])

    }

    for ; i < word1Len; i++ {
        mergedString += string(word1[i])
    }

    for ; j < word2Len; j++ {
        mergedString += string(word2[j])
    }

    return mergedString
}

While doing the problem I found out there are some really cool ways to use for in Golang. As I come from C like languages and Java background a while loop has been pretty much ingrained into my body. I do think Go has done a remarkable job capturing the essence of while loops with a for style loop.

Notice how the declaration can be done independently outside of for construct, and how both variables i,j can be incremented together. Definitely impressed with the construct. string on the other hand gave me pretty much a headache. I think the problem is that I started asking Gemini. rune, byte, slice of runes and a lot of things I tried resulted in failure. I'm sure this will get easier with time as I learn the internals, but for the time I will still keep my opinion that strings in golang are hell.
