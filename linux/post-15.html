Improving my Hyprland WM setup

Feb 01, 2025, Posted in Linux

In my quest for building a perfect minimalist linux system, I decided to not use a waybar. I believe it wastes a lot of screen space and doesn't really give a minimalist vibe. While waybar does look pretty sexy, I think it also wastes a lot of space. Here is how my waybar looked like

You'll notice it has everything, a battery monitor (fish icon), cpu usage or memory usage stats, current time/date etc. It has a workspace switcher too, it was useful for me to know which is the current (1..9) workspace. It all looked cool, but I think these docks are inspired by mac, and I didn't like the idea of wasting screen space. Think about it when you're actually working on something, how often do you look on time or the topmost panels, I reckon rarely.

So I decided to ditch waybar completely from the configuration and instead use hyprexpo which is a hyprland plugin. I built the latest version of hyprland from git using yay and installed this hyprexpo plugin.

The next thing I missed was having a notification system. But dunst is a good way to achieve notifications too. Here is a sample notification and the entire configuration can be seen in my dotfiles. .

My first application during startup is wtfutil which will create a small dashboard on workspace 1, so I can read hackernews there, or check memory and cpu usage and also do stuff like keeping todo notes. It's very extensible and has its own configuration file.

Overall I'm quite satisfied with how my system looks now.
